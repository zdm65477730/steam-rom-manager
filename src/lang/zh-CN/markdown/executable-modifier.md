# 可执行文件修饰符 `[支持变量]`{.noWrap}

如果未设置，默认为 `"${exePath}"`{.noWrap}。此字段可用于在可执行文件前后添加所需字符，这些字符将被添加到Steam快捷方式的`目标`属性中。例如，假定 `${exePath}`{.noWrap} 为 `C:\RetroArch\retroarch.exe`，您可以通过将`可执行文件修饰符` "cmd" 设置为以下内容来最小化启动`Retroarch`：

```
"cmd" /k start /min "${exePath}"
```

在这种情况下，`目标`属性将以以下内容开头：

```
"cmd" /k start /min "C:\RetroArch\retroarch.exe"
```

后面跟着您在`命令行参数`字段中选择的任何启动参数。

您可以使用任何其他变量来构建最终的可执行文件。

此设置会影响Steam的APP ID。

## 快捷方式透传

如果您启用`跟随.lnk/.desktop到目标`，并且通配符搜索找到一个`.lnk`或`.desktop`文件，即一个快捷方式，那么`${filePath}`变量将包含快捷方式的目标，而不是指向快捷方式的路径。如果您想要添加可执行文件参数，请将其添加到快捷方式的目标中，或在解析器中使用`命令行参数`字段。
